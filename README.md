# 🤖 AI Imposter

AI Imposter is a real-time multiplayer party game where players answer funny questions and try to detect which answer was generated by AI.

Each round includes answers from real players and one hidden AI-generated answer.  
Players vote for the answer they think was written by AI, earn points, and compete across 5 rounds on a live leaderboard.

---

## 🎮 How It Works

Players join the game with a nickname and enter a shared lobby.

When all players are ready, the game starts automatically:

1. A funny question is shown.
2. Each player submits an answer.
3. The system generates one AI answer.
4. All answers are displayed anonymously.
5. Players vote for the answer they think was written by AI.
6. The AI answer is revealed.
7. Points are calculated and the leaderboard updates.

After 5 rounds, the game ends with a final podium.

## 🧠 Game Rules

| Rule | Value |
|---|---|
| Players | 2–9 |
| Rounds | 5 |
| Answer time | 20 seconds |
| Voting time | 10 seconds |
| Reveal time | 6 seconds |

Players cannot vote for their own answer.  
Invalid or missing answers cannot receive votes.

---

## 🏆 Scoring

| Action | Points |
|---|---:|
| Correctly voting for the AI answer | +2 |
| Another player votes for your answer | +1 |
| Voting for a human answer | 0 |

The AI does not receive points and does not appear in the leaderboard.

## 🏗️ Architecture

AI Imposter is built as a frontend-first real-time application.

Supabase is used as the main source of truth for the game state.  
React and MobX manage the client-side experience, while Supabase Realtime keeps all players synchronized during the game.

Main database tables:

- game_state
- players
- questions
- answers
- votes

The project separates the UI, state management, Supabase communication, and AI logic into different layers to keep the code organized and maintainable.

---

## 🔐 Authentication

Players do not need to create an account.

They join using only a nickname, while Supabase Anonymous Auth runs behind the scenes to connect each browser session to a player in the database.
